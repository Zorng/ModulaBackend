# Attendance Module

**Responsibility:** Staff shifts, check-in/out, GPS policy

## Structure

- `api/` - HTTP routes and controllers
- `app/` - Use cases (check-in, check-out)
- `domain/` - Shift, Attendance entities
- `infra/` - Attendance repository
- `migrations/` - Attendance-related database tables
- `tests/` - Module tests

## Key Features

- Staff check-in/check-out
- Shift tracking
- GPS validation (Phase 2)
- Attendance reports
