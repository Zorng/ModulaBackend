# Implementation Decisions (ADRs)

This folder stores ADR-style documents for implementation choices that should not immediately mutate the knowledge base.

Use this when:

- the decision is still evolving
- the decision is a pragmatic constraint (delivery, performance, dependency)
- the team needs a stable reference while implementation proceeds in parallel

When a decision becomes stable “product truth”, promote it into:

- `knowledge_base/BusinessLogic/...`

Template:

- `_implementation_decisions/ADR_TEMPLATE.md`
